# Forecast Model

霞候使用可解释规则模型评估朝霞和晚霞机会。模型目标不是给出精确气象结论，而是把摄影观察决策需要的要素拆开：能不能被照到、有没有画布、能不能看清、会不会被降水或低云挡住。

## Core Formula

```text
总分 =
光路 / 低云遮挡 25
+ 中高云画布 25
+ 降水与能见度 20
+ 气溶胶散射 10
+ 风向趋势 10
+ 天文窗口与现场修正 10
```

最终总分会转成经验概率，同时生成大片火烧云概率和烧度分级。

## Solar Events

支持两类事件：

- `sunrise`: 朝霞，以日出为中心，重点看东方光路。
- `sunset`: 晚霞，以日落为中心，重点看西方光路。

事件时间来自 Open-Meteo daily `sunrise` / `sunset`，不是固定时间。

## Light Corridor

光路采样使用郑州中心点加两个方向走廊点：

```text
郑州中心: 0.55
近端走廊: 0.30
远端走廊: 0.15
```

方向由太阳方位角计算。晚霞看西方光路，朝霞看东方光路。

低云是主要遮挡项：

- `0-20%`: 光路干净
- `20-50%`: 需要看云缝
- `>50%`: 明显受阻
- `>75%`: 对总分设上限

## Canvas

中高云是霞光画布。

高云：

- `25-75%`: 最优
- `10-25%`: 偏少
- `75-90%`: 偏满
- `<10%` 或 `>90%`: 扣分

中云：

- `10-50%`: 最优
- `50-70%`: 递减
- `>70%`: 扣分

高云过满时，模型会降低大片火烧云概率，但保留粉灰云幕可能性。

## Visibility And Precipitation

能见度：

- `>12km`: 好
- `8-12km`: 一般
- `5-8km`: 风险
- `<5km`: 强扣分并封顶

降水和降水概率会明显拉低总分。日出 / 日落窗口有明显降水时，整体概率会被封顶。

## Aerosol

气溶胶既可能加色，也可能造成灰霾。

- AOD `0.15-0.45`: 暖色散射友好
- PM2.5 `15-45 μg/m³`: 轻微加色
- PM2.5 `45-75`: 扣通透度
- PM2.5 `>75`: 强扣

dust 只在能见度较好时小幅加暖色；能见度差时不加分。

## Wind

风向按气象风向处理，即“风从哪里来”。

如果风从日出 / 日落方向来，且风速在 `4-25km/h`，则方向走廊云况对郑州更有参考价值：

- 方向走廊开口、低云少：加分
- 方向走廊低云或雨墙重：扣分

阵风 `>40km/h` 会降低稳定性。微风且 PM/AOD 偏高时增加霾风险。

## Burn Grade

烧度分级用于摄影决策：

- `大烧`: 低云少、画布合适、能见度好，大片概率高。
- `中烧`: 条件较好，但范围、低云或通透度有短板。
- `小烧`: 有局部上色机会，依赖临场云缝。
- `弱烧`: 可能只有淡粉、淡黄或灰粉云幕。
- `难烧`: 低云、降水或通透度触发硬风险。

烧度分级和总体概率有关，但不会只看总体概率。低云、降水、能见度会触发硬性下调。

## Manual Signals

应用提供现场修正按钮：

- 地平线有清晰云缝
- 云边已经发亮
- 云底开始泛黄
- 低云灰墙压死
- 整片乳白高云幕

人工修正幅度会限制在 `-15` 到 `+15`，避免单一现场信号完全覆盖模型判断。
